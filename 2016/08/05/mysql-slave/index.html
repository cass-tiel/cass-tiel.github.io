MySql主从同步配置

发表于 2016-08-05   |   分类于 linux ， mysql

摘要

MySql-AB复制是一种数据复制技术

  • MySQL提供的一种高性能、高可用的解决方案。
  • 复制
    • 单向复制：不要在slave上做写入操作，会导致主从数据不一致。
    • 双向复制
    • 异步复制
  • ab复制的模式
    • 一主一从
    • 一主多从
    • 双主
    • 多主多从

一、AB复制原理

主服务器必须开启二进制日志

  • 1）首先master将更新写入二进制日志，有一个二进制日志索引文件跟踪日志循环
  • 2）slave通过I/O线程向master索要二进制日志，slave知道从哪个binlog点开始索要
  • 3）master接收到slave的I/O请求之后，根据slave提供的位置点开始传输日志
  • 4）slave收到日志后，写入本地的relay log中
  • 5）slave通过SQL线程读取relay log，在数据库中执行相应的操作，至此，
    主从数据保持一致，从服务器进入等待状态，等待主服务器的下一次更新。

二、环境说明

  • 操作系统：CentOS 6.7 64位
  • MySQL数据库版本：mysql-5.5.35
  • master：172.16.0.100
  • slave：172.16.0.101

三、配置步骤：

注：一些基础环境配置需要自行完成

1、master端配置

1）修改配置文件

2）重启mysql服务

3）授权一个用来实现复制的用户

4）做一次全库备份

5）将备份传到slave服务器

2、slave的配置

1）修改配置文件

2）停掉数据库(全新库不用做)

3）初始化数据库(全新库不用做)

4）恢复来自master的数据

3、在master上查看当前日志文件及位置点

4、在slave上手动同步数据

注意：当slave是start状态的时候，是不能够执行change的

5、master服务器解锁

6、验证
